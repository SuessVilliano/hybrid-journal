# Hybrid Journal — Full Code Audit (2026-06-10)

Scope: entire `src/` plus `base44/functions/`. Focus: numerical accuracy and data
integrity, since traders need to trust this app daily. Findings are verified
against the code (file:line cited). Severity: **Critical** = corrupts data or
shows materially wrong numbers; **High** = wrong numbers in common scenarios or
broken core flows; **Medium** = wrong in edge cases or misleading; **Low** = polish.

Already fixed on this branch: the floating AI chat button overlapped the mobile
bottom tab bar in app mode (hid the right-most tab). It now sits above the tab
bar with safe-area support, and the chat panel opens fully on-screen on phones
(`src/Layout.jsx`, `src/components/ai/FloatingAIAssistant.jsx`).

## CRITICAL — fix before promoting daily use

### 1. Broker sync fabricates random trades and balances
`base44/functions/syncBroker/entry.ts:179-214`
Every broker except Binance (Kraken, Tradovate, MT4/5, cTrader are routed here
with `supportsAutoSync: true`) returns `generateSimulatedTrades(broker_id, 20)`
plus a random account balance (10000–15000), which are persisted as real trades.
The `broker_trade_id` is `Date.now()`-based so dedupe never matches — **every
sync adds 20 more fake trades**, and AutoSyncManager syncs hourly. The frontend
LLM-fabrication path was already deprecated for this exact reason
(`brokerAPIHelper.jsx:113-122`), but the backend still does it.
**Fix:** return "broker not supported" instead of simulated data; never persist
simulated balances.

### 2. Binance sync can't work and invents P&L
`base44/functions/syncBroker/entry.ts:141-176`
`/api/v3/myTrades` is called without the required HMAC signature (always 401).
If it ever succeeded: P&L is invented as ±1% of notional, entry==exit price,
balance hardcoded 10000, only BTCUSDT fetched.
**Fix:** implement HMAC SHA256 signing; compute realized P&L from matched fills
or don't store P&L.

### 3. "Delete Import & Trades" deletes zero trades
`src/pages/Imports.jsx:44-57`
Deletion filters on `t.import_source?.includes(importId)`, but no code ever
stores the Import id in `import_source` (parsers set constants like
`'CSV Import'`). The UI confirms "all N associated trades permanently deleted"
while every trade remains; re-importing then duplicates everything.
**Fix:** stamp `import_id: importRecord.id` on each created trade and delete by
that key (paginated).

### 4. CSV re-import silently duplicates every trade
`src/components/trading/ImportModal.jsx:156-162`
No dedupe key and no existence check (unlike `statementIngest`). Combined with
#3, the natural "delete and re-import" flow doubles all trades, corrupting
balances, win rate, and equity curves.
**Fix:** dedupe on ticket id (or symbol+entry_date+side+qty+pnl fallback).

### 5. Currency parsing corrupts imported P&L
`src/components/utils/tradeParsers.jsx:48-62, 99-104, 147-150, 325-377`;
also `base44/functions/syncHybridFunding/entry.ts:92`
`parseFloat` is applied to raw statement values: `"1,234.56"` → **1.23**,
`"1 234.56"` → 1, `"$500"` → NaN, `"(250.00)"` → NaN. The validity guard
(`trade.pnl !== undefined`) lets NaN through, poisoning totals app-wide.
**Fix:** a shared `parseMoney()` (strip `$`/commas/spaces, parentheses =
negative, reject non-finite) used by every parser; reject NaN rows visibly.

### 6. MT4 HTML and CSV parsers disagree on the Profit column
`src/components/utils/tradeParsers.jsx:48-62` (HTML uses cells[12]=profit,
[11]=swap) vs `:102-104` (CSV uses values[13]=pnl, [12]=swap). Against the
standard MT4 layout (Commission 10, Taxes 11, Swap 12, Profit 13) the HTML
parser stores **swap as the trade's P&L**. MT4 dates (`2024.01.15 10:30`) also
fail `new Date().toISOString()` per-row, silently dropping trades.
**Fix:** locate columns by header name, not fixed index; explicit date format
parsing.

### 7. Futures position sizing uses tick value as point value → ~4x oversized
`src/components/risk/RiskCalculator.jsx:24, 48-53`
`Math.abs(entry - sl)` is in points but is multiplied by `$5` (NQ's per-**tick**
value; 1 point = 4 ticks = $20). A trader risking "1%" actually risks ~4%.
Also hardcoded to NQ — ES, GC, CL etc. are all wrong.
**Fix:** central instrument spec table (tick size, tick value, point value,
contract multiplier) shared with TradeForm auto-P&L and the backtester.

### 8. Backtester runs on LLM-generated (fictional) price data
`src/components/backtesting/BacktestEngine.jsx:5-42`
"Historical" OHLCV is generated by `InvokeLLM` ("Generate realistic price
movements") while the UI advertises "Real-time historical data"
(`Backtesting.jsx:153`). `optimizeParameters` (378-403) regenerates a
*different* random series per parameter combo, so optimization rankings are
meaningless.
**Fix:** real market-data source (broker API or CSV); fetch once and reuse the
same series across optimization combos.

### 9. Strategy conditions executed as raw JavaScript
`src/components/backtesting/BacktestEngine.jsx:200`
`new Function(...)(...)` on user-entered text — code injection, and `null`
indicator values silently coerce to 0 (see High #8).
**Fix:** sandboxed expression parser (e.g. expr-eval) with explicit null checks.

### 10. Drawdown chart computed backwards in time, divides by zero
`src/components/analytics/AdvancedCharts.jsx:22-36`
Trades arrive newest-first (`Trade.list('-entry_date')`) and are never sorted,
so the cumulative/drawdown curve runs in reverse. `peak` starts at 0 so early
losses produce `Infinity`/`NaN` rendered in the chart. Drawdown is also a % of
cumulative P&L peak rather than account equity.
**Fix:** sort ascending by close date; use `initial_balance + peak` as the
denominator; guard `peak <= 0`.

### 11. "Public" pages bounce anonymous visitors to Landing
`src/Layout.jsx:56-80` vs `src/App.jsx:27`
App.jsx whitelists Landing, PublicDashboard, Pricing, SocialFeed, Help, etc. as
public, but Layout runs `base44.auth.me()` on every mount and on failure
redirects everything except Landing. Pricing/Help/SocialFeed are unreachable
when logged out.
**Fix:** skip the auth/onboarding check for public pages.

## HIGH

1. **Expired token = blank /login lockout** — `src/App.jsx:47-55` returns
   `null` whenever auth is required, including on `/login` itself; with a stale
   token the user can never reach the login form. Render auth routes and clear
   the stale token.
2. **Admin gating is a hardcoded email check in client code** —
   `src/Layout.jsx:210` (`email !== 'liv8ent@gmail.com'`) only hides nav items;
   admin pages (BrokerConnections, Backtesting, SharedAccess, AdminMessaging)
   are reachable by URL by any logged-in user. Gate on `user.role === 'admin'`
   and enforce server-side; remove the personal email from source.
3. **React Query v5 misuse** — (a) `useQuery` `onSuccess` was removed in v5, so
   the new-signal browser notification feature never fires
   (`src/pages/LiveTradingSignals.jsx:82-94`); (b) ~70 call sites use
   `invalidateQueries(['key'])` (array form), which in v5 matches **every**
   query — each mutation triggers an app-wide refetch storm. Use
   `invalidateQueries({ queryKey: ['key'] })`.
4. **Encrypted credentials never decrypted in sync paths** —
   `syncDXTrade/entry.ts:177`, `syncBroker/entry.ts:138` send the `enc:v1:...`
   blob as the password once `SECRET_VAULT_KEY` is set; only `syncCrossTrade`
   decrypts. Every DXTrade sync then fails.
5. **Login passwords stored in plaintext** — `BrokerConnectionForm.jsx:120-155`
   encrypts only `api_key`/`api_secret`; `password`, `username`,
   `platform_username` persist raw while the UI claims "encrypted and stored
   securely". Encryption also fails open (plaintext saved on failure,
   `:150-152`). Encrypt password too; fail closed.
6. **Synced trades have no `account_id`** — all server sync paths
   (`syncDXTrade:244-262`, `syncBroker:70-86`, `statementIngest:86-107`) create
   trades without `account_id`, so Accounts-page balance/win-rate
   (`Accounts.jsx:68`) silently excludes every broker-synced trade.
7. **Concurrent sync race duplicates trades** — `AutoSyncManager.jsx:11-45`
   rebuilds all timers every 60s (query refetch changes array identity; any
   `sync_interval` > 60s never fires via interval), `BrokerConnections.jsx`
   fires `onLoginSync` on the same page, and no sync path holds a lock —
   overlapping read-then-create duplicates trades. Add a per-connection
   check-and-set lock.
8. **Backtest enters trades during indicator warmup** —
   `BacktestEngine.jsx:238` hardcodes warmup at 50 bars; longer-period
   indicators are `null`, which coerces to 0 in condition evaluation, so
   `close > SMA_200` fires spurious trades. Also: equity curve ignores
   unrealized P&L (understates max drawdown) and open positions at the last bar
   are silently discarded (`:317, 334, 348-354`).
9. **Profit factor = 0 when there are no losing trades** — four independent
   copies (`Dashboard.jsx:135`, `ComprehensiveStats.jsx:30`,
   `PublicDashboard.jsx:47`, `BacktestEngine.jsx:345`) all report PF 0 (styled
   red / failing the "Profitable Strategy" badge) for a flawless record, while
   `HybridScoreWidget.jsx:26` caps it at 5 — the same data shows different PFs
   on the same screen. Show "∞"/N-A consistently from one shared helper.
10. **In-place sort mutates the React Query cache** —
    `DetailedCharts.jsx:31-32`, `PublicDashboard.jsx:65-66` call
    `trades.sort(...)` on the cached array, reordering data for every other
    consumer. Copy first: `[...trades].sort(...)`.
11. **Daily summaries show the wrong day off-UTC** —
    `TradingSummaries.jsx:11, 14-39` parses `"YYYY-MM-DD"` as UTC midnight then
    mutates it in local time; users west of UTC get the previous day's range.
12. **Weekly consistency metric merges weeks across months** —
    `HybridScoreWidget.jsx:38` computes week-of-month keyed only by year, so
    Jan-W2/Feb-W2/... collapse into one bucket; a third of the Hybrid Score is
    computed over ~6 meaningless buckets. Use ISO weeks.
13. **Forex sizing floors to whole lots and breaks JPY pairs** —
    `RiskCalculator.jsx:56-58`: $100 risk / $500-per-lot → `Math.floor(0.2)` =
    **0 lots, "Actual Risk $0.00"**; pip multiplier `*10000` is 100x off for
    JPY pairs. Also the Calculators-page instance gets `accountBalance = 0` and
    never computes anything (`Calculators.jsx:65`, `RiskCalculator.jsx:9,36`),
    and the "Stocks" option has no calculation branch at all.
14. **Direction inference defaults everything to Short** —
    `tradeParsers.jsx:319-322`: any header containing "type" matches (an
    order-type column of Market/Limit makes every trade Short); any
    non-buy/long value (including `balance` rows) becomes Short instead of
    being rejected. Same pattern in the cTrader and MT4 HTML parsers.
15. **Statement dedupe merges distinct partial fills** —
    `statementIngest/entry.ts:77` fallback key `symbol_entryTime_pnl` collides
    for split fills (second fill silently lost); side stored as raw `Buy/Sell`
    against the `Long|Short` enum; `pnl_net` double-counts fees when commission
    is reported negative.
16. **Date parsing: no timezone, MM/DD assumed** — all parsers use
    `new Date(...)` in browser-local time on server-time statements (MT4 EET);
    `03/04/2025` is always read as March 4. Day attribution (calendar, daily
    P&L, prop-firm daily loss) shifts near midnight. `timezoneHelper.jsx`
    exists but is unused by parsers.
17. **AI position-size suggestion written into the risk-% field** —
    `PositionSizeCalculator.jsx:46` assigns the AI's *position size as % of
    portfolio* into *risk per trade %* — different quantities; silently changes
    user risk and can exceed the slider's range.

## MEDIUM

- **Expectancy charges break-evens as losses** — `ComprehensiveStats.jsx:178`
  uses `(100 - winRate)` as the loss probability; BE trades are counted at
  `avgLoss`. Use `losses/total`.
- **Two win-rate definitions in the app** — Dashboard/ComprehensiveStats/
  Accounts use `wins/total`; PerformanceMetrics/Strategy/Instrument widgets use
  `wins/(wins+losses)` (and render `NaN%` for all-BE groups). Same trades →
  different win rates on different screens. One shared helper.
- **Open trades mixed into realized stats** — no stats path filters by
  `exit_date`/`trade_status`; `TradeCalendar.jsx:168` can throw on null `pnl`;
  `ComprehensiveStats.jsx:331, 371` hardcode "Open trades: 0" and "0R".
- **Prop-firm card wrong on both axes** — `PropFirmRulesCard.jsx:38-40`:
  `Math.abs(todaysPnl)` counts a *profitable* day against the loss limit, and
  the % divides by account balance, not the limit; "trailing" drawdown is a
  static % of current balance, not trailing the high-water mark.
- **Gross P&L fee-sign error** — `ComprehensiveStats.jsx:19`:
  `totalPnl + Math.abs(totalFees)` adds swap credits instead of subtracting;
  fee signs are mixed between TradeForm (positive cost) and MT4 imports
  (negative).
- **"Avg Realized R-Multiple" is not R** — `ComprehensiveStats.jsx:374` is
  payoff ratio with a `Math.max(avgLoss, 1)` clamp that distorts micro-lot
  accounts.
- **Public dashboard privacy mode leaks dollars** — `PublicDashboard.jsx:290,
  341`: `(pnl/100)*100` is just `pnl` rendered with a `%` suffix — wrong number
  AND defeats `hideDollars`; best-trade always gets a `+` prefix.
- **Backtest fills ignore gaps** — `BacktestEngine.jsx:286-300` fills stops/TPs
  at the level even when the bar gaps through it; no slippage/commission. RSI
  (`:122-123`) is lagged one bar and non-Wilder, so thresholds won't match
  charts.
- **Compound calculator shows one random Monte-Carlo path** —
  `CompoundCalculatorWidget.jsx:32-58` (`Math.random()` in `useMemo`) re-rolls
  per keystroke instead of the closed-form expectation; `tradesPerDay = 0` →
  `i % 0` → NaN chart and "-100% return".
- **Locale-dependent daily grouping** — `DetailedCharts.jsx:82, 90-92` keys
  days by `toLocaleDateString()` then re-parses; dd/mm locales get Invalid
  Date and arbitrary bar ordering. Use ISO `yyyy-MM-dd` keys.
- **Broker panel joined on wrong key** — `Accounts.jsx:76` compares
  `bc.account_number === accountId` (internal id) — "Broker Balance / Last
  Synced" essentially never renders.
- **`last_sync` vs `last_sync_at` schism** — schema says `last_sync_at`;
  syncDXTrade/syncBroker/AutoSyncManager use `last_sync`; statementIngest and
  ConnectionStatusCard use `last_sync_at` → "Last Sync: Never"/Invalid Date
  depending on path, and the 5-minute login-sync throttle misses ingest syncs.
- **Excel accepted but unparseable** — `ImportModal.jsx:235` accepts
  `.xlsx/.xls` then runs CSV parsing on the binary. Integrate a real parser or
  drop it from the accept list.
- **PDF/AI import fabricates missing values** — `tradeParsers.jsx:575-579`
  defaults missing dates to *now*, price to 0, qty to 1, stored as real data;
  no cross-check of P&L sum vs statement total.
- **Import "View Analysis" shows wrong trades** — `ImportVisualization.jsx:20-24`
  matches by filename (never set) OR any same-day trade.
- **Onboarding duplicates TraderProfile** — `Onboarding.jsx:83` always creates;
  `Layout.jsx:63-64` reads `profiles[0]` unsorted, which can re-trigger the
  onboarding redirect.
- **Unbounded/truncated fetches** — `Accounts.jsx:34-37` default page size;
  Dashboard/Trades/Analytics cap at 1000 with no truncation warning — stats
  silently wrong for active traders. Paginate or aggregate server-side.
- **PWA install layer broken** — `index.html:5-8` links `/manifest.json` which
  doesn't exist (no `public/` dir); title is "Base44 APP", external favicon, no
  theme-color/apple-touch-icon.
- **Layout re-fetches auth + profile on every navigation** —
  `Layout.jsx:56-80` (deps `[currentPageName]`), duplicating AuthContext, and
  uses `window.location.href` (full reload) for intra-app redirects.
- **`handleSync` clobbers concurrent updates** — `BrokerConnections.jsx:64-86`
  spreads a stale `...connection` into the update and writes undefined
  `account_balance`.
- **CopyMonitor unhandled rejections** — `CopyMonitor.jsx:11-29` async poll
  every 10s with no try/catch.
- **`detectFormat` misroutes CSVs** — `tradeParsers.jsx:424`: any file
  mentioning "ticket" gets MT4 fixed-index parsing.
- **Auth token in localStorage** — `src/lib/app-params.js:22-33` persists
  `access_token` permanently (XSS-exfiltratable, survives logout).

## LOW (selected)

- `parseFloat(x) || default` snaps cleared inputs back to defaults
  (`RiskCalculator.jsx:115`, `PropFirmRulesCard.jsx:95,106`); cleared numeric
  fields save NaN (`Accounts.jsx:441`, `BrokerConnectionForm.jsx:539`).
- CSV export rows unquoted (commas break columns) and locale-ambiguous dates
  (`ExportMenu.jsx:45-57`).
- Header rows split with naive `split(',')` while data rows use the
  quote-aware parser (`tradeParsers.jsx:9`); no RFC-4180 `""` escape handling.
- Parser `platform` strings violate the Trade schema enum — if enforced, every
  MT4/5 row fails creation after "successful" parsing.
- Equity/drawdown ordered by `entry_date`; realized P&L should book at
  `exit_date`.
- `BacktestResults.jsx:73` XAxis bound to nonexistent `trade` field;
  `Backtesting.jsx:54` crashes if all optimization combos fail;
  `sharpe_ratio: 0` hardcoded and stored.
- Pie chart of per-instrument `pnl` can't represent negative slices; "Top
  Symbols" renders losers as green `+$-123.45` (`PerformanceMetrics.jsx`).
- `Dashboard.jsx:314` hides the Avg Loss card when avgLoss is exactly 0.
- Debug logs of user emails/payloads in production paths
  (`LiveTradingSignals.jsx`, `AutoSyncManager.jsx`); query errors swallowed and
  returned as `[]`, masking outages as "no data".
- StrictMode commented out (`main.jsx:7-9`); `react-hooks/exhaustive-deps` not
  enabled; ESLint doesn't cover `src/lib`, `src/api`, `App.jsx`.
- TradeForm auto-P&L `(exit-entry)×qty` ignores contract multipliers/pip values
  (`TradeForm.jsx:232`) and recalculates over a manually typed `0`.
- Prop-firm settings created only via button onClick (Enter key skips them) and
  never linked to the account.
- Zero test coverage: no test script, runner, or test files anywhere.
- 2.2 MB single JS bundle (no code splitting / route-level lazy loading).

## Enhancement roadmap (highest trust-per-effort first)

1. **Shared metrics module** (`src/lib/metrics.js`): single implementation of
   winRate, profitFactor, expectancy, drawdown, daily grouping, and an
   `isClosed(trade)` predicate — used by Dashboard, Analytics, PublicDashboard,
   Accounts, exports, and AI prompts. Eliminates the entire
   "same-data-different-numbers" class.
2. **`parseMoney()` + explicit date parsing** (date-fns format strings, source
   timezone setting) in one place for all importers/sync functions; reject
   non-finite values with visible per-row errors.
3. **Instrument spec table** (tick size/value, point value, pip decimals,
   contract multipliers) shared by RiskCalculator, TradeForm auto-P&L, and the
   backtester.
4. **Import pipeline hardening**: server-side parsing (single code path with
   statementIngest), dedupe keys, `import_id` stamping, preview-before-commit,
   and post-import reconciliation (sum of P&L vs statement total).
5. **Kill all simulated data paths** in syncBroker; implement real broker APIs
   or clearly disable unsupported brokers.
6. **React Query v5 conformance pass** (70 invalidateQueries call sites, dead
   onSuccess, mutation onError toasts) + error boundary + StrictMode +
   exhaustive-deps lint.
7. **Server-enforced roles** replacing the hardcoded admin email; encrypt all
   broker credentials, fail closed.
8. **Tests**: vitest + RTL starting with the pure logic (metrics, parsers,
   RiskCalculator, BacktestEngine) — these are the trust-critical functions and
   the easiest to test.
9. **Real backtest data + realism**: cached historical series, same series
   across optimization combos, gap-aware fills, slippage/commission inputs,
   mark-to-market drawdown, Sharpe/Sortino.
10. **PWA completion**: real manifest + icons + theme-color (the tab-bar/safe-
    area work on this branch already handles the layout side).
